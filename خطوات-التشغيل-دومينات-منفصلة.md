# تشغيل المنصة — الباك والفرونت على دومينات مختلفة

## الفكرة باختصار

| جزء      | دوره              | مثال رابط       |
|----------|-------------------|------------------|
| **الباك اند** | API + لوحة تحكم   | `https://api.موقعك.com` |
| **الفرونت اند** | واجهة اللاعبين   | `https://ctf.موقعك.com` |

الفرونت يرسل طلبات الـ API إلى الباك اند، لذلك:
1. **الباك** يحتاج يعرف من وين يجيه الفرونت (CORS) → `FRONTEND_ORIGIN`
2. **الفرونت** يحتاج يعرف وين الباك → `VITE_API_BASE_URL`

---

## 1) الدومينات (Catch the Hilal)

| الجزء | الدومين |
|-------|---------|
| **الفرونت اند** (واجهة اللاعبين) | `https://ctf.jordancyberclub.com` |
| **الباك اند** (API + لوحة تحكم) | `https://ctf-admin.jordancyberclub.com` |

---

## 2) تشغيل الباك اند

```bash
cd ~/ctf-platform/backend
cp .env.example .env
nano .env
```

**في ملف `.env` اكتب:**

```env
SECRET_KEY=السلسلة_العشوائية_من_الأمر_اللي_تحت
FRONTEND_ORIGIN=https://ctf.jordancyberclub.com
```

لتوليد `SECRET_KEY`:

```bash
openssl rand -hex 32
```

انسخ الناتج وضعها مكان `السلسلة_العشوائية_من_الأمر_اللي_تحت`.

**مهم:** `FRONTEND_ORIGIN` لازم يكون `https://ctf.jordancyberclub.com` بالضبط.

ثم شغّل الباك اند:

```bash
docker compose up -d --build
```

---

## 3) تشغيل الفرونت اند

```bash
cd ~/ctf-platform/frontend
cp .env.example .env
nano .env
```

**في ملف `.env` اكتب:**

```env
VITE_API_BASE_URL=https://ctf-admin.jordancyberclub.com
```

**مهم:** بدون `/` في النهاية.

ثم ابنِ وشغّل الفرونت:

```bash
export VITE_API_BASE_URL=https://ctf-admin.jordancyberclub.com
docker compose up -d --build
```

## 4) مثال بالـ IP (بدون دومين)

| المتغير           | القيمة                    |
|-------------------|---------------------------|
| `FRONTEND_ORIGIN` | `http://209.74.79.65:4173` |
| `VITE_API_BASE_URL` | `http://209.74.79.65`    |

- الباك: `http://209.74.79.65` (منفذ 80)
- الفرونت: `http://209.74.79.65:4173`

## 6) ربط الدومينات بالـ VPS

لازم الاثنين يشيروا لنفس الـ IP (`209.74.79.65`):

- `ctf.jordancyberclub.com` → A record → `209.74.79.65`
- `ctf-admin.jordancyberclub.com` → A record → `209.74.79.65`

وتحتاج **Nginx أو Caddy** على الـ host لتوجيه الطلبات. لتجنب تعارض المنافذ، غيّر الباك ليعمل على 8080:

في `backend/docker-compose.yml` غيّر `80:80` إلى `8080:80` في خدمة nginx.

ثم الـ reverse proxy يوجّه:
- `ctf.jordancyberclub.com` → `localhost:4173` (الفرونت)
- `ctf-admin.jordancyberclub.com` → `localhost:8080` (الباك)

ممكن استخدام **Cloudflare** للـ DNS والـ HTTPS، أو إعداد Nginx يدوياً مع Let's Encrypt.

## ملخص سريع

| مكان الإعداد | الملف        | المتغير              | القيمة        |
|--------------|--------------|----------------------|---------------|
| الباك اند   | `backend/.env`  | `FRONTEND_ORIGIN`    | رابط الفرونت |
| الفرونت اند | `frontend/.env` | `VITE_API_BASE_URL` | رابط الباك اند |

بعد أي تغيير في `VITE_API_BASE_URL` لازم تعيد بناء الفرونت: `docker compose up -d --build`
